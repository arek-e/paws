Protected Agent Workspace Sandboxes

Zero-trust credential injection for AI agents. Secrets never enter the sandbox, they're injected at the network layer by a per-VM TLS proxy.

curl -fsSL https://getpaws.dev/install.sh | bash

Works on any VPS. No Cloudflare or DNS provider needed. Domain optional.

Zero secrets in the VM

API keys, tokens, and credentials never enter the sandbox. A per-VM TLS proxy injects them at the network layer. If the VM is compromised, there's nothing to steal.

Sub-second boot

Firecracker memory snapshots restore VMs in under a second. Each session gets a fresh, ephemeral microVM with its own network namespace and proxy.

Port exposure

Agents run fullstack apps, and users access them via Pangolin tunnel URLs. Per-port access control: SSO, PIN, or email whitelist. Share links for quick access.

Agent harness

Claude Code pre-installed. Just provide a prompt and constraints. The workload script is auto-generated. More agent frameworks coming soon.

Snapshot templates

6 built-in templates: minimal, Node.js, Python, Docker, fullstack, Claude Code. Or build your own from the dashboard.

Dashboard

Fleet overview, session streaming, daemon management, snapshot builder, tunnel admin, user management, SSO config. All in one place.

How it works

Agent in VM: curl https://api.anthropic.com/v1/messages | +-- VM resolves api.anthropic.com (real IP) +-- iptables DNAT rewrites dest to per-VM proxy +-- Proxy terminates TLS, reads SNI | +-- Domain in allowlist? | YES: inject x-api-key header, forward to real API | NO: drop connection (TCP RST) | +-- Agent sees normal HTTPS response Never saw the API key. Not in env, not in memory, not on disk.
